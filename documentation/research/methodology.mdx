# Research Methodology

Mutual Dissent is designed as a research platform, not just a debate tool. Every debate produces a structured JSON transcript that enables systematic analysis of cross-vendor AI behavior.

## What the Transcripts Capture

- **Per-round, per-model responses** — track how each model's position evolves across reflection rounds
- **Routing provenance** — know exactly which provider handled each call
- **Token and cost data** — control for response length when analyzing outputs
- **Experiment metadata** — link debates to campaigns, conditions, and finding references
- **Ground-truth scores** — quantify synthesis accuracy against known answers

## Experiment Design

Use `ExperimentMetadata` to tag debates for structured analysis:

```python
from mutual_dissent.models import ExperimentMetadata
from mutual_dissent.orchestrator import run_debate
from mutual_dissent.config import load_config

config = load_config()

meta = ExperimentMetadata(
    experiment_id="study-2-safety-attrition",
    source_tool="manual",
    condition="baseline",
    variables={"rounds": 1, "panel": "default"},
)

transcript = await run_debate(
    "Your research query",
    config,
    rounds=3,
)
transcript.metadata["experiment"] = meta
```

## Cross-Tool Integration

Mutual Dissent integrates with CounterSignal and CounterAgent as a research platform:

**CounterSignal (IPI)** — inject adversarial payloads into panelist context via `panelist_context`. Observe whether poisoned context alters model responses, comparison behavior, or synthesis outcomes.

**CounterSignal (RXP)** — surface RAG-retrieved documents as panelist context. Validate whether retrieval-poisoned documents influence cross-vendor consensus.

**CounterAgent** — use debate as a detection oracle. Run debates with and without injected MCP tool outputs to measure influence on synthesis.

## Transcript Analysis

Use `dissent list` and `dissent show` to review transcripts. For bulk analysis, load transcripts directly from `~/.mutual-dissent/transcripts/` as JSON — the schema is documented at [Transcript Schema](/schemas/transcript).

```bash
# Export a debate to markdown for review
dissent show abcd1234 --verbose --output markdown --file analysis.md

# Pull all transcripts for a batch study
ls ~/.mutual-dissent/transcripts/
```
